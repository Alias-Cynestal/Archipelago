# Instructions pour Control Archipelago

## Installation

Pour jouer à Control dans le mode Archipelago, il faut quelques étapes:

1. Créer le fichier de paramètres pour le jeu. Pour faire cela, vous devez installer le fichier control.apworld dans le dossier custom_worlds de votre installation du launcher Archipelago. À partir de là, il sera possible d'utiliser l'outil de création de paramètres intégré du Launcher pour déterminer les paramètres de votre session d'Archipelago. Pour plus d'informations sur les paramètres disponibles, voir [Paramètres](#paramètres-disponibles).
2. Si vous êtes l'hôte de votre session d'Archipelago, vous devez générer votre monde et lancer le serveur Archipelago. Je n'expliquerai pas cela ici, mais vous pouvez vous fier aux instructions disponibles sur le [site officiel Archipelago](https://archipelago.gg/tutorial/Archipelago/setup_en).
3. Pour que l'Archipelago fonctionne, le client Ap.Control et le patch Ap.Control.Patcher est nécessaire. Pour plus d'informations, voir ce [lien](https://github.com/Alias-Cynestal/ArchipelagoControlClient).
4. Terminé ! Si tout est bien configuré, alors vous devez pouvoir jouer sans aucun problème!



## Paramètres disponibles
- __Enable Sector Unlocks et Enable Clearance Level Unlocks__: ils sont mis comme paramètres, mais je ne sais pas s'ils marchent correctement ou non. Pour l'instant, je conseillerais de les garder activés.
- __Enable Progressive Clearance Level Level Unlocks__: Permet de choisir la progression du niveau d'accès. Mis à vrai, le niveau augment en fonction du nombre d'améliorations que vous obtenez. À faux, vous pouvez obtenir n'importe quel niveau, n'importe quand.
- __* Percentage__: Permet de choisir le pourcentage des items dans les items de remplissage.

D'autres seront ajoutés éventuellement, et je vous prie de me faire part de vos idées!

## J'ai trouvé un bug, que dois-je faire ?
Il faut séparer les bugs en deux catégories.
- S'il s'agit d'un bug en lien avec la génération du monde ou bien un problème de logique dans l'ordre d'obtention des items, alors le problème relève de ce projet. Pour cela, vous pouvez créer un billet ici et j'y jetterai un œil.
- S'il s'agit d'un bug en lien avec tout le reste (les items ne sont pas bien donnés, les locations ne sont pas détectées, etc.), ça relève plutôt du client. Veuillez créer un billet sur la page du projet de client plutôt que celle-ci.
- Peu importe le cas, si vous préférez, il existe un thread pour Control sur le serveur Discord AP After Dark. Vous pouvez envoyer votre problème là-bas, je le regarde régulièrement.

Dans tous les cas, si vous avez des questions ou vous voulez contribuer au projet, n'hésitez pas à me contacter !
